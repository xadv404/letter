Crawling multi-domaines

Entrée

  • Fichier .txt — un domaine par ligne
  • Centaines de domaines en parallèle
  • Reprise via crawl.state.json

Contrôles

  • Profondeur : 1 – 10 niveaux
  • Limite : 10 – 500 pages / domaine
  • Workers dynamiques + pause / resume
  • Timeout 5 min par domaine

Stabilité

  • Throttling adaptatif CPU / RAM
  • NORMAL → CRITICAL (délais ×4)
  • Export incrémental live sur disque

Scoring SQLi des paramètres

Tier Score Exemples Action
HIGH ≥ 85 id, search_term, filter_by Accepté + URL flaggée
MEDIUM 65 – 84 SecLists match, params inconnus plausibles Accepté pour dorks
LOW 50 – 64 page, sort, category Exclu par défaut
EXCLUDE < 50 utm_source, sidebar, session tokens Ignoré 
URL : /products?category=electronics&id=42

category → LOW (58)   → rejeté
id       → HIGH (95)  → accepté, URL dans targets_*.txt
